## `commit` — Thin Commit & Push

**Type:** skill or command  
**Trigger:** `/commit`

A thin dispatcher that commits and pushes by delegating to the right skill for the context, with a self-contained fallback.

### Dispatch order

1. **`/commit-commands:commit`** — use when the `commit-commands` plugin is active (detected by checking `$CLAUDE_PROJECT_DIR/.claude/settings.json` for the plugin, or by attempting to invoke and catching absence).
2. **`/ecc:prp-commit`** — use when a PRP (`docs/prp.md` or `.claude/prp/`) is present in the working tree (signals a PRP-driven workflow).
3. **Fallback** — when neither is available:
   - `git add` the files referenced in the current conversation context (changed files, files mentioned by the user).
   - `git commit -m "<auto-generated>"` — message derived from the changed files and conversation context: `<type>(<scope>): <what changed>`.
   - `git push` if the branch has a remote tracking ref (`git rev-parse --abbrev-ref @{u}` succeeds).

### Dependencies

- `git` on PATH (always required)
- `/commit-commands:commit` plugin (optional — detected at runtime)
- `/ecc:prp-commit` skill (optional — detected at runtime)
